# 1104 行動バイアス設計分析（CryptoSignal AI™）

本ドキュメントは、損失局面の投資家行動に関する行動経済学・心理学の知見を、CryptoSignal AI™ の機能・UI・運用に落とし込むための設計メモである。

## 1. 主要バイアスと設計対策
- 損失回避と参照点依存（プロスペクト理論）
  対策: リスク％上限・ハードSL・自動サイズ計算を既定。損失領域での倍プッシュをブロック。
- ディスポジション効果
  対策: 事前SLと段階利確（1R/2R等）をデフォルト。ストップ外し禁止。
- サンクコスト効果
  対策: 金額よりR・％表示を優先し、情動的固執を弱める。
- 確証バイアス
  対策: 各シグナルで賛否要因（反証条件）を1行併記。
- 情報カスケード／ハーディング
  対策: 群集過熱フラグ（ソーシャル急騰×価格乖離）を表示。
- マイオピック損失回避
  対策: 通知のバッチ化とTTL/再評価時刻の明示で短期評価の反射行動を抑制。
- 過剰売買／過信
  対策: 回転率・手数料比の可視化、連続損失ロックとクールダウン。

## 2. シグナル必須フィールド（Telegram/ダッシュボード共通）
- Invalidation（無効化価格）
- Risk Unit（1R=資産×risk_pct）
- Risk %（推奨0.5–1.0）
- Position Size（自動算出）
- TTL / Re-eval（有効期限・再評価時刻）
- Counter-signal（反証条件）
- Crowd Heat（カスケード注意: Low/Med/High）
- Base Rate（30D命中率/平均R/最大DD）

## 3. ガードレール（Non-Functional）
- リスク上限: 1%/取引、3%/日（超過はブロック）
- 連続損失ロック: N連敗でクールダウン
- 通知設計: バッチ配信＋評価頻度の推奨ガイド
- 準拠監査: SL遵守・サイズ逸脱・クールダウン遵守の週次計測

## 4. KPI と公開指標
- シグナル別ベースレート、平均R、最大DD
- コンプライアンス指標（SL遵守率、サイズ逸脱率、回転率）
- 連敗後の介入効果（ロック導入前後の勝率・DD比較）

## 5. Phase 0 実装範囲（2025-11-04）
- メッセージテンプレの項目追加（上記必須フィールド）
- 日次損失上限・連続損失ロック（最小構成）
- 週次統計配信（ベースレート/平均R/遵守率）
